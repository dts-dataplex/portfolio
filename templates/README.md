# Templates Directory

This directory contains template files that serve as examples and starting points for the career portfolio management system.

## Purpose

Templates provide:
- **Example structure**: Reference implementations without PII
- **Starting points**: Copy templates to create your private data files
- **Documentation**: Working examples of data formats
- **Version control**: Maintained in git as framework documentation

## Files

### portfolio.template.json
Example career data structure showing all available fields.

**Usage:**
```bash
# Create your portfolio from template
cp templates/portfolio.template.json portfolio.json
# Edit portfolio.json with your information
```

**Contains examples of:**
- All 16 top-level sections
- Privacy visibility controls
- Cross-referenced data (work → compensation)
- Organizational linking (promotions)
- Export profile configurations
- Tagging and categorization

### data_quality_report.md
Template for Career Data Manager quality reports.

**Generated by:** Career Data Manager agent
**Output location:** `output/reports/DATA_QUALITY_REPORT.md`

**Contains:**
- Validation status
- Data completeness metrics
- Cross-reference verification
- Skills categorization summary
- Recommendations

## Agent Templates

The `agents/` subdirectory contains template versions of agent definitions.

### agents/*.md
Reference implementations of the 6 career management agents.

**Files:**
- `job-market-analyst.md`
- `career-strategy-advisor.md`
- `resume-generator.md`
- `cover-letter-writer.md`
- `application-tracker.md`
- `career-data-manager.md`

**Purpose:**
- Documentation of agent specifications
- Version-controlled reference implementations
- Starting point for agent customization
- Backup of agent definitions

**Active vs Template:**
```
templates/agents/*.md  →  Reference (version-controlled)
.claude/agents/*.md    →  Active (used by Claude Code)
```

## Using Templates

### Portfolio Data

1. **Copy template:**
   ```bash
   cp templates/portfolio.template.json portfolio.json
   ```

2. **Edit with your data:**
   - Replace placeholder values
   - Remove unused sections
   - Add your work history, skills, education
   - Configure privacy settings

3. **Validate:**
   ```bash
   python3 scripts/validate.py
   ```

### Agent Definitions

1. **Review template:**
   ```bash
   cat templates/agents/resume-generator.md
   ```

2. **Customize if needed:**
   - Copy to `.claude/agents/`
   - Modify responsibilities or workflows
   - Adjust communication protocols

3. **Test with Claude Code:**
   ```bash
   @resume-generator help
   ```

## Template Maintenance

### Updates
When updating templates:

1. **Preserve generic data**: Never add real PII to templates
2. **Update both locations**: Keep `templates/agents/` and `.claude/agents/` in sync
3. **Validate examples**: Ensure JSON validates against schema
4. **Document changes**: Note significant updates in commit messages

### Best Practices

- Use realistic but fake data in examples
- Include comments explaining optional vs required fields
- Show diverse examples (different industries, roles, levels)
- Demonstrate all features (privacy controls, linking, tags)

## Privacy Protection

**Templates are safe for git:**
- No real names, emails, phone numbers
- No actual company names (use "Acme Corp", "Example Inc")
- No real addresses or locations (use "San Francisco, CA")
- No actual salary numbers (use example ranges)
- No real project details (use generic descriptions)

**Real data is excluded:**
- portfolio.json (actual career data)
- output/ (generated materials)
- input/ (source resumes and job postings)
- .agent-system/runtime data

## Validation

All JSON templates should validate against schemas:

```bash
# Validate portfolio template
python3 -c "import json; from jsonschema import validate; \
validate(json.load(open('templates/portfolio.template.json')), \
json.load(open('schemas/portfolio.schema.json')))"
```

When adding new templates:

3. **Include examples**: Show diverse use cases
4. **Validate**: Test against schemas
5. **Document**: Add to this README
6. **Commit**: Safe to add to git
